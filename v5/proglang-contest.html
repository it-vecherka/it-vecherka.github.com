Четвертый выпуск

Давайте языками померяемся

Вот только что попалась на глаза статья Лутца Претшельта "Эмпирическое сравнение 7 языков программирования". Статья годная и маленькая, к прочтению рекомендую.

Сравнение языков программирования в ней делается на основе довольно небезынтересной задачи.

Кодирование номера телефона.

Сопоставить телефонному номеру фразу. На вход подаётся словарь, для примера можно взять вот этот. Потом нужно принять телефонный номер и вернуть все возможные фразы, которыми его можно закодировать. Кодирование идёт на основе букв, которые традиционно пишут на клавишах телефона:

{ "2" => "ABC", "3" => "DEF", "4" => "GHI", "5" => "JKL", "6" => "MNO", "7" => "PQRS", "8" => "TUV", "9" => "WXYZ" }

Задача - найти в словаре все комбинации слов, которыми может быть закодирован этот телефонный номер. Например, для номера "63263379355" одна из возможных кодировок - "me codes well", всего же возможных расшифровок 57.

Эта задача, в отличие от многих, здесь загаданных, не имеет хитрого читерского алгоритма, позволяющего быстро прийти к успеху. Конечно, и здесь можно натупить, но, скорее всего, к правильному решению вы придёте быстро. Интересно помериться количеством строк, изящностью, читаемостью и скоростью работы/потреблением памяти. Чем больше здесь будет разных языков, тем лучше. Приглашаются джависты, скалисты, рубисты, джаваскриптизёры и кофескриптизёры, кложа и хаскель, питон, баш/zsh и с(++), ну и, конечно, Кутейко.

Подготовьте программу к получению текстового файла, каждая строка которого - номер телефона, и пусть выводом программы будет "номер - возможная кодировка", т.е.,

номер1 возможная_кодировка1
номер1 возможная_кодировка2
номер1 возможная_кодировка3
номер2 возможная_кодировка1

для того, чтобы можно было проверить под нагрузкой.

Вот репозиторий, в который нужно сдать решение: https://github.com/dmitriy-kiriyenko/proglang-contest.

Защити честь родного / тайно любимого языка!
